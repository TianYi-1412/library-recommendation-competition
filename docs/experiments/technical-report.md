# 技术报告：图书馆图书推荐系统

> 比赛：全球校园人工智能算法精英大赛——图书推荐赛道

---

## 1. 赛题概述

### 任务
根据图书馆历史借阅数据，预测每个用户下一本可能借阅的图书。

### 数据
- **88,378** 条借阅记录
- **1,451** 个用户
- **58,549** 本图书
- 特征：用户 demographics、图书元数据、借阅时间戳、续借次数

### 评估
- **指标**：F1 Score（精确率和召回率的调和平均数）
- **目标**：每个用户输出 1 本推荐图书
- **有效门槛**：F1 > 0.0055

---

## 2. 解决方案架构

我们实现了 **8 大算法家族**，组织为模块化流水线：

```
数据预处理
    +-- 特征工程（28 维）
    +-- 图构建

算法家族
    +-- 1. XGBoost + GNN 流水线 (v8)      [GBDT + 图特征]
    +-- 2. GNN 模型                        [LightGCN、多关系 GNN]
    +-- 3. 轻量 Transformer                 [自注意力打分]
    +-- 4. GRU4Rec                         [序列 RNN]
    +-- 5. RecBole 框架                     [8 模型集成]
    +-- 6. Item2Vec + LightGBM             [嵌入 + 排序]
    +-- 7. DeepFM                          [深度因子分解机]
    +-- 8. 多模态双塔                      [M3E 文本 + ID 嵌入]

集成与后处理
    +-- 智能加权融合
    +-- 多样性增强
    +-- 续借感知调整
```

## 3. 关键技术

### 3.1 多路召回（12 路）

| 通道 | 说明 |
|------|------|
| 二级分类 | 用户偏好的图书分类 |
| 院系分类 | 院系级热门分类 |
| 关键词匹配 | 核心关键词重叠 |
| 作者偏好 | 之前借过的作者 |
| 出版社偏好 | 偏好的出版社 |
| 细化分类 | 细粒度分类 |
| 复合分类 | 复合分类 |
| 主题标签 | 主题内容匹配 |
| 图相似度 | GNN 嵌入相似度 |
| 热门图书 | 时间衰减热度 |
| Item2Vec 相似度 | 共借模式 |
| Apriori 规则 | 关联规则挖掘 |

### 3.2 特征工程（28 维）

- **行为特征**：续借分数、热度分数
- **内容匹配**：分类匹配、关键词匹配、作者/出版社匹配
- **交叉特征**：院系-分类、关键词-分类交互
- **图特征**：用户-图书嵌入点积
- **类别特征**：17 维哈希桶编码

### 3.3 时间衰减

```
weight = exp(-days_since_interaction / half_life)
```

- 半衰期：GRU4Rec 用 30 天，XGBoost 用 90 天
- 近期交互权重更高

### 3.4 兴趣强度

```
intensity = min(borrow_duration_normalized + repeat_weight, 1.0)
```

- 重复权重：每次额外借阅 +0.3（上限 0.6）
- 量化用户-图书关系强度

### 3.5 智能集成权重

四阶段权重计算：
1. **性能**：0.5*recall + 0.3*NDCG + 0.2*precision
2. **多样性**：高多样性模型奖励
3. **质量**：覆盖率 + 新颖性奖励
4. **特殊规则**：Pop 模型惩罚、低多样性惩罚

## 5. 实验结果

### 最佳单模型

| 模型 | F1 分数 | 说明 |
|------|---------|------|
| GRU4Rec | **0.0158** | 最佳序列模型 |
| XGBoost+GNN (v8) | 0.0150+ | 最佳特征模型 |
| Item2Vec+LightGBM | 有竞争力 | 强基线 |

### 集成提升

- 单最佳模型：F1 ~0.0158
- 集成融合：通过多样性进一步提升
- 续借感知调整：对高续借用户额外提升

## 6. 洞察

### 有效的做法

1. **序列模型（GRU4Rec）**：时序模式是强信号
2. **兴趣强度**：区分交互强弱有帮助
3. **多路召回**：更多候选 = 更好覆盖
4. **丰富特征**：28 维特征向量捕获复杂模式
5. **智能集成**：多样化模型互补

### 效果不佳的做法

1. **随机负样本**：难负样本挖掘可以改进
2. **短序列**：seq_len=3 限制长期模式捕获
3. **哈希冲突**：哈希特征可能有冲突
4. **文本嵌入**：M3E 慢且内存密集

### 关键结论

- **混合方法**（特征工程 + 深度学习）表现最好
- **时序建模**对借阅预测至关重要
- **集成多样性**比单个模型性能更重要
- **领域知识**（校历、续借行为）提供强信号

## 7. 未来改进

1. **难负样本挖掘**：采样更难负样本用于训练
2. **更长序列**：增加序列长度以更好建模时序
3. **注意力机制**：特征交互的自注意力
4. **图增强**：向 GNN 添加更多关系类型
5. **AutoML**：自动化特征选择和超参调优
6. **多任务学习**：联合预测借阅 + 续借 + 评分
